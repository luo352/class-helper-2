# 班级助手2.0 - H5替代方案技术文档

## 一、方案概述

### 1.1 背景
由于微信小程序存在兼容性问题，用户无法稳定访问班级助手系统。本方案提供基于H5的替代访问方案，确保用户能够通过持久化途径（固定链接、可保存的二维码）便捷进入系统。

### 1.2 方案目标
- 提供稳定的H5访问入口，替代微信小程序
- 支持多端访问（iOS、Android、Windows、Mac）
- 确保所有核心功能完整实现且正常运行
- 满足性能要求（页面加载≤3秒，操作响应≤1秒）


### 3.3 静态部署方案

#### 方案一：Nginx部署
```nginx
server {
    listen 80;
    server_name your-domain.com;
    root /path/to/dist;
    index index.html;

    # 支持SPA路由
    location / {
        try_files $uri $uri/ /index.html;
    }

    # 静态资源缓存
    location ~* \.(js|css|png|jpg|gif|svg)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
}
```

#### 方案二：云平台部署

##### CloudStudio 部署（推荐）
1. 登录 CloudStudio (https://cloudstudio.net/)
2. 创建工作空间，选择前端环境（Node.js）
3. 上传项目代码或从Git仓库克隆
4. 执行构建命令：`npm run build:h5`
5. 使用内置预览功能或配置静态服务
6. 获取公网访问地址

##### Vercel 部署
1. 登录 Vercel (https://vercel.com/)
2. 导入项目 Git 仓库
3. 配置构建命令：`npm run build:h5`
4. 配置输出目录：`dist`
5. 部署完成后获得访问域名

##### Netlify 部署
1. 登录 Netlify (https://www.netlify.com/)
2. 连接 Git 仓库
3. 配置构建命令：`npm run build:h5`
4. 配置发布目录：`dist`
5. 部署完成后获得访问域名

##### GitHub Pages 部署
1. 创建 gh-pages 分支
2. 配置 GitHub Actions 自动构建
3. 构建产物推送到 gh-pages 分支
4. 访问地址：`https://username.github.io/repository/`

## 四、访问方式

### 4.1 本地访问
- **地址**: `http://localhost:10086/`
- **适用场景**: 开发调试、本地测试

### 4.2 局域网访问
- **地址**: `http://[本机IP]:10086/`
- **适用场景**: 团队协作测试、手机扫码访问

### 4.3 公网访问
- **地址**: `https://your-domain.com/`（需配置域名）
- **适用场景**: 正式上线、用户访问

### 4.4 二维码访问
- **二维码页面**: `http://localhost:10086/qr-code.html`（开发环境）
- **二维码页面**: `https://your-domain.com/qr-code.html`（生产环境）
- **功能**: 自动生成访问二维码，支持移动端扫描
- **特性**: 
  - 纯前端实现，不依赖外部CDN或服务
  - 支持复制访问链接
  - 支持保存二维码图片到本地
  - 自动适配当前部署地址

## 十、故障处理

### 10.1 常见问题
| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 页面白屏 | JS加载失败 | 检查网络连接、清除缓存 |
| 定位失败 | 浏览器权限未授权 | 引导用户开启定位权限 |
| 样式错乱 | CSS加载顺序问题 | 检查CSS引入顺序 |
| 路由404 | 服务器配置问题 | 配置SPA路由重定向 |

### 10.2 日志记录
